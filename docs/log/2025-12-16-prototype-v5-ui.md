# Prototype V5 UI Design

Reference documentation for the Trip Kiwi prototype v5 interface.

## Overview

A three-panel trip planning interface with resizable panels and theme support.

- **Left Navigation** - Trip list and app controls
- **Content Panel** - Plans/Sources list and editor
- **Chat Panel** - AI assistant conversations

## Components

### Left Navigation

**Expanded state:**
- Logo and app name (Trip Kiwi)
- Collapse toggle button
- New Trip button
- My Trips section with trip list
- Active trip highlighted
- Settings dropdown at bottom

**Collapsed state:**
- Narrow icon rail (52px)
- Expand toggle button
- New Trip icon button
- Settings icon button
- Tooltips on hover

### Trip Header

- Spans across Content + Chat panels
- Editable trip name (inline text input)
- Actions dropdown menu:
  - Rename Trip
  - Trip Settings
  - Delete Trip (destructive)

### Content Panel

**View picker dropdown:**
- Plans (itinerary items)
- Sources (saved places, resources)

**List view:**
- 2-column card grid
- Cards show title + subtitle
- Muted background style (bg-muted/70)
- Click to open detail view

**Detail view:**
- Breadcrumb navigation (e.g., "Plans > Taipei")
- Editable title input
- TipTap rich text editor
- Actions dropdown (Delete)
- Delete confirmation dialog

### Chat Panel

**Header:**
- Chat picker dropdown with search
- New chat button
- Actions menu (Rename, Delete)

**Messages area:**
- User messages: right-aligned, muted background
- Assistant messages: left-aligned, plain text
- Scrollable message history

**Input area:**
- Multi-line textarea
- Send button (disabled when empty)

## Theming

**Color modes:**
- Light
- Dark
- System (follows OS preference)

**UI Scale:**
- 90%
- 95%
- 100% (default)
- 105%
- 110%

Settings accessible from:
- Left nav Settings dropdown (expanded)
- Settings icon (collapsed nav)

## Panel Sizing

| Panel | Default | Min | Max |
|-------|---------|-----|-----|
| Left Nav | 18% | 14% | 28% |
| Chat Panel | 40% | 25% | 60% |

All panels are resizable with drag handles.

## Visual Style

- Muted card backgrounds for items
- Consistent padding (p-4 content areas, p-2 toolbars)
- Border separators between sections
- Ghost button variant for most actions
- Destructive styling for delete actions (red text)
